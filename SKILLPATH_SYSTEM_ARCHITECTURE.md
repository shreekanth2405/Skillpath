# 🗺️ SkillPath AI System Architecture & Walkthrough

This document provides a comprehensive step-by-step analysis of the SkillPath AI platform, covering the Frontend, Backend, Navigation, and AI components.

---

## 🏗️ 1. Frontend Architecture (React + Vite)
The frontend is a single-page application (SPA) focused on immersive, gamified user experiences.

### Key Components:
- **Entry Points**: `main.jsx` and `App.jsx`.
- **Global State**: Manages JWT authentication and gamification metrics (XP, Level, Coins).
- **Protected Layout**: Uses a wrapper component (`ProtectedLayout`) to guard routes and redirect unauthenticated users to `/login`.
- **Module Hubs**:
    - **Learning Hub**: Course catalogs and AI-generated skill paths.
    - **Career Hub**: Resume tools, Kanban job tracker, and market analytics.
    - **Games Hub**: Immersive coding challenges (3D/2D).
    - **Communication Hub**: Language learning and AI speech assistance.

### Data Sync Flow:
1. `App.jsx` checks `localStorage` for `auth` token.
2. If authenticated, it fetches user data from `${API_URL}/users/profile`.
3. Global state (`userXP`, `userCoins`) is updated and passed down to the `Header` and `Dashboard`.

---

## ⚙️ 2. Backend Architecture (Node.js + Express)
The backend is a RESTful API designed for security and scalability.

### Core Structure:
- **Server Initialization**: `server.js` starts the process; `app.js` configures middle-wares.
- **Security Middle-wares**: Includes `helmet`, `cors`, `xss-clean`, and `express-rate-limit`.
- **Database (Prisma + PostgreSQL)**:
    - **Schema**: Defines models for `User`, `Job`, `Resume`, `Habit`, `Event`.
    - **Client**: `prismaClient.js` provides a centralized instance for database operations.
- **End-points**:
    - `/api/auth`: Login/Register.
    - `/api/users`: Profile updates.
    - `/api/v1/jobs`: Kanban-based job tracking operations.
    - `/api/v1/habits`: Consistency monitoring.

## 🧭 3. Navigation System
The platform uses a dynamic navigation system that adapts to the user's current activity.

### Modes:
1. **Public Mode**: Minimal navbar for guest users.
2. **Platform Mode**: Full header with solutions megamenu, XP stats, and notification center.
3. **Focus Mode (Minimal)**: Triggered during games or assessments (`isMinimal = true`). Removes all distractions to keep the user focused on the challenge.

---

## 🤖 4. AI & Automation Services
Standalone Python services handle computationally expensive or specialized tasks.

- **AI Matcher (`ai_matcher.py`)**: Uses NLP (TF-IDF + Cosine Similarity) to calculate the relevance of scraped jobs to a user's skills.
- **Scraper Engine (`scraper.py`)**: Designed to crawl career pages and populate the database with fresh opportunities.
- **RAG Integration**: The frontend allows users to upload documents directly into the `Header`, which stores context in a `knowledgeBaseRef` for the AI Chatbot.

## 🔄 5. End-to-End Data Flow Example
**Scenario: Synchronizing Game Rewards**
1. User completes a challenge in **Code Escape House**.
2. Frontend calls `syncXP` endpoint with earned rewards.
3. Backend `dashboardController.js` updates the user's XP and calculates if a "Level Up" is required.
4. Database is updated via Prisma.
5. Success response updates the frontend global state, reflecting the new level and coins in the `Header` immediately.
